# 🧭 技术导师 Mentor SOP

**一句话目标**：让参与者快速上手，并在截止前完成可交付作品。

> 小型活动可由 1 人兼任技术导师 Mentor（教学演示 + 技术支持）；大型活动再拆分。

## 现场执行清单（活动当天）

### 活动前（T-3 ~ T-1）
- [ ] 准备共学文档（工具链接/账号/提示词/分工建议）。
- [ ] 录制或准备 5-10 分钟视频教程（现场翻车可切换）。
- [ ] 测试现场网络与工具可用性，准备替代方案/镜像。
- [ ] 建立作品提交表并测试权限。
- [ ] 与主持人/中控对齐时间节点与报时方式。
- [ ] 将 PPT/教程文件备份到本地。

### 活动当天（开场与演示）
- [ ] 15 分钟内完成教学演示，字体放大、步骤可复制。
- [ ] 明确“最小可行路径”，告诉大家先做完再优化。
- [ ] 演示后 5 分钟内开始巡场。

### 巡场辅导（14:45 - 17:00）
- [ ] 每 15-20 分钟巡一圈，逐组询问卡点。
- [ ] 给出下一步动作，帮助他们收敛目标。
- [ ] 按 60/30/10 分钟节点提醒收尾与提交。
- [ ] 处理账号/工具/网络问题，必要时切换替代方案。

### 作品提交（16:30 - 17:00）
- [ ] 检查分享权限，协助上传/填表。
- [ ] 核对提交数量与名单，补齐缺失信息。

### 活动后
- [ ] 收集共性问题，更新教程与提示词。

## 注意事项与应急

### 关键标准
- 主动巡场 > 被动答疑。
- 先保证“能交付”，再追求“更完美”。
- 视频教程是保险，演示不稳定时直接切换。

::: tip 大型活动可拆分
可拆为教学讲师、巡场导师、提交/统计三岗。
:::

### 常见问题与应急处理
| 问题 | 解决方案 |
| --- | --- |
| **工具无法访问** | 切换镜像/替代工具，或提供共享账号 |
| **队伍进度卡住** | 帮其缩小目标，先完成 MVP |
| **作品链接打不开** | 检查分享权限或重新生成链接 |
| **提交混乱** | 统一口径，设定唯一入口与截止时间 |
